# LangGraph Parallel Workflow Documentation

This document explains the code implemented in `pararrel_workflow.ipynb`. The workflow demonstrates advanced LangGraph concepts, including parallel node execution (Fan-Out), joining parallel executions (Fan-In), and conditional cyclic logic (Feedback Loops).

## 1. State Definition (`GraphState`)

```python
class GraphState(TypedDict):
    topic: str
    web_research: str
    wiki_research: str
    draft: str
    style_review: str
    tech_review: str
    grammar_review: str
    editor_feedback: str
    final_article: str
    revision_count: int
```

**Explanation:**
In LangGraph, the `State` represents the memory of the workflow. Every node receives this dictionary, reads from it, and returns updates to it.

- `topic`: The initial input provided by the user.
- `web_research` & `wiki_research`: Store the output of the parallel research nodes.
- `draft`: Stores the text generated by the Writer node.
- `*_review`: Stores the individual critiques from the parallel reviewer nodes.
- `editor_feedback`: Holds construction criticism if the draft is rejected.
- `final_article`: Holds the final polished text when approved.
- `revision_count`: Tracks how many times the draft has been rejected to prevent infinite loops.

## 2. Parallel Research Nodes (Fan-Out 1)

```python
def web_search_node(state: GraphState):
    results = web_tool.invoke(state['topic'])
    return {"web_research": results}

def wiki_search_node(state: GraphState):
    results = wiki_tool.invoke(state['topic'])
    return {"wiki_research": results}
```

**Explanation:**
These two functions run completely independently. Because they are both connected from the `START` node in the graph builder, LangGraph will execute them at the exact same time. This drastically reduces the time spent waiting for API calls.

## 6. The Conditional Edge (The Feedback Loop)

```python
def check_quality(state: GraphState):
    if state.get("final_article"):
        return "finalize"
    elif state.get("revision_count", 0) >= 2:
        return "finalize" # Force completion to avoid infinite loops
    else:
        return "revise"
```

```python
# In the Graph Builder:
builder.add_conditional_edges(
    "editor",
    check_quality,
    {"revise": "writer", "finalize": "finalize_node"}
)
```

**Explanation:**
This is the core of the Agentic loop. Instead of a hardcoded path (`add_edge`), we use `add_conditional_edges`. LangGraph runs the `check_quality` function after the `editor_node` finishes.

- If `check_quality` returns `"revise"`, the graph routes _backwards_ to the `writer_node`, initiating a new cycle (but this time, the Writer has access to `state['editor_feedback']`).
- If it returns `"finalize"`, it moves towards the end of the graph.

## 7. Graph Compilation and Execution

```python
graph = builder.compile()
final_state = graph.invoke(initial_state)
```

**Explanation:**
`.compile()` freezes the graph structure so it can be executed. `graph.invoke()` is the single entry point. You pass in the initial state dictionary, and LangGraph manages all the parallel execution, waiting, and routing automatically, ultimately returning the fully populated final state when the `END` node is reached.
